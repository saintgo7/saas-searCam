## 2. 사용자 인터뷰 시나리오

### 2.1 인터뷰 설계

| 항목 | 내용 |
|------|------|
| 대상 | 20~40대 여성/남성, 숙박 시설 이용 경험자, 1인 가구 |
| 인원 | 12~15명 (페르소나별 2~3명) |
| 방법 | 1:1 심층 인터뷰 (60분) + 프로토타입 사용성 테스트 (30분) |
| 장소 | 온라인 화상 또는 오프라인 (카페/사무실) |
| 보상 | 카페 기프티콘 1만원 |

|---|------|------|

| A4 | 몰래카메라를 확인하기 위해 실제로 무언가를 해본 적이 있나요? (육안 확인, 앱, 장비 등) | 기존 행동 패턴 |

## 3. 상세 페르소나

### 3.1 페르소나 1: 김서연 (여행자)

| 항목 | 내용 |
|------|------|
| **이름** | 김서연 |
| **나이** | 28세 |
| **직업** | 대기업 마케팅 담당자 |
| **거주지** | 서울 마포구 |
| **기술 수준** | 중상 (앱 설치/사용 익숙, 기술 원리는 모름) |
| **연소득** | 4,200만원 |
| **스마트폰** | Galaxy S24 |

#### 상황

국내외 여행을 즐기며, 에어비앤비를 주로 이용합니다. 최근 SNS에서 숙소 몰래카메라 피해 사례를 접한 후 불안해졌습니다. 여행 전 숙소 리뷰를 꼼꼼히 확인하지만, 도착 후 확인할 방법이 마땅하지 않습니다.

#### 페인포인트

- 에어비앤비 체크인 후 혼자 방에 들어갔을 때 느끼는 불안감
- 어디를 어떻게 확인해야 하는지 모름
- 기존 탐지 앱을 깔아봤지만 오탐이 너무 많아서 삭제함
- 전문 탐지 장비는 비싸고, 여행 때마다 가져갈 수 없음

#### 니즈

- 체크인 직후 1분 이내에 빠르게 점검하고 싶음
- 결과를 쉽게 이해하고 싶음 (기술 용어 아닌 "안전/위험" 판단)
- 오탐으로 여행 기분이 망가지지 않았으면 함

#### 하루 일과 (여행일)

```
14:00  에어비앤비 체크인
14:05  짐 풀기
14:10  ★ SearCam Quick Scan 실행 (30초)
14:11  결과 확인 → "안전" → 안심
14:15  샤워 준비
14:30  외출
```

#### 앱 사용 시나리오

1. 체크인 직후 앱을 열고 Quick Scan 버튼을 탭합니다
2. 30초간 Wi-Fi 스캔이 진행되는 동안 짐을 풉니다
3. "안전 12/100" 결과를 확인하고, 발견된 기기(공유기, TV 등) 목록을 봅니다
4. "Full Scan 하기" 버튼이 보이지만, Quick 결과에 만족하여 "완료"를 탭합니다
5. 만약 "주의" 이상이 나왔다면, 렌즈 찾기 모드로 추가 확인합니다

| **거주지** | 서울 관악구 원룸 |
| **기술 수준** | 상 (IT 종사자) |
| **연소득** | 3,200만원 |
| **스마트폰** | iPhone 15 |

#### 상황

최근 원룸에 입주했습니다. 부동산 중개를 통해 구했지만, 이전 세입자나 집주인이 카메라를 설치했을 수 있다는 불안이 있습니다. 공중화장실 이용 시에도 항상 주변을 확인하는 습관이 있습니다.

#### 페인포인트

- 원룸 입주 시 전문 점검 서비스를 부르기에는 비용(15~30만원)이 부담
- 화장실, 탈의실 이용 시마다 반복되는 불안감
- 어디를 확인해야 하는지 구체적인 가이드가 없음
- 기존 앱: 기술 수준이 높아서 오탐인 줄 알면서도 불안함

#### 니즈

- 입주 시 1회 정밀 스캔 + 이후 빠른 반복 스캔
- 기술적 근거를 볼 수 있으면 좋겠음 (MAC 주소, 포트 정보 등)
- 오탐 원인을 이해할 수 있는 상세 설명

#### 하루 일과

```
08:00  기상, 샤워
08:30  출근 준비
09:00  출근 (지하철)
12:30  점심 → 회사 화장실 이용 시 습관적으로 확인
18:30  퇴근
19:00  피트니스 (탈의실 이용)
19:05  ★ SearCam Quick Scan (피트니스 탈의실 Wi-Fi)
21:00  귀가
```

#### 앱 사용 시나리오

1. 원룸 입주 첫날 Full Scan을 실행합니다 (3분)
2. 렌즈 찾기 모드로 천장, 벽면, 연기 감지기 등을 직접 확인합니다
3. 체크리스트(숙소)를 따라 육안 점검까지 완료합니다
4. 리포트를 PDF로 저장하여 기록을 남깁니다
5. 이후 피트니스 탈의실에서는 Quick Scan만 빠르게 실행합니다

### 3.3 페르소나 3: 박성호 (출장 직장인)

| 항목 | 내용 |
|------|------|
| **이름** | 박성호 |
| **나이** | 42세 |
| **직업** | 중견기업 영업부장 |
| **거주지** | 경기 수원시 |
| **기술 수준** | 중 (카카오톡, 네이버 등 일상 앱 사용) |
| **연소득** | 6,500만원 |
| **스마트폰** | Galaxy S23 |

#### 상황

월 2~3회 지방 출장을 가며, 모텔이나 비즈니스호텔을 이용합니다. 뉴스에서 숙박업소 몰래카메라 관련 기사를 접한 후 신경이 쓰이기 시작했습니다. 기술에 관심은 있지만 복잡한 앱은 싫어합니다.

#### 페인포인트

- 피로한 출장 후 복잡한 점검은 하기 싫음
- "버튼 하나만 누르면 끝"이어야 함
- 기술 용어가 나오면 바로 이탈
- 유료 앱에 대한 거부감은 적지만, 효과가 확실해야 함

#### 니즈

- 극도로 단순한 사용 경험 (앱 열기 → 버튼 1개 → 결과)
- 결과를 색상과 이모지로 직관적으로 이해
- 사용 이유를 동료에게 설명할 수 있을 정도의 쉬운 앱

#### 하루 일과 (출장일)

```
07:00  KTX로 이동
10:00  거래처 미팅
12:00  점심
14:00  추가 미팅
18:00  모텔 체크인
18:05  ★ SearCam Quick Scan 실행 → 30초 → "안전" 확인
18:10  샤워 후 휴식
```

#### 앱 사용 시나리오

1. 모텔 체크인 후 앱을 열면 큰 "Quick Scan" 버튼이 보입니다
2. 한 번 탭하고 30초 기다립니다
3. "안전" 초록색 화면을 보고 안심합니다
4. Full Scan이나 상세 기능은 사용하지 않습니다

### 3.4 페르소나 4: 최은지 (피트니스 이용자)

| 항목 | 내용 |
|------|------|
| **이름** | 최은지 |
| **나이** | 31세 |
| **직업** | 프리랜서 디자이너 |
| **거주지** | 서울 강남구 |
| **기술 수준** | 중상 |
| **연소득** | 4,800만원 |
| **스마트폰** | iPhone 16 Pro |

#### 상황

매일 헬스장 탈의실과 샤워실을 이용합니다. 탈의실에는 Wi-Fi가 없는 경우가 많아 네트워크 스캔이 불가능합니다. 물리적 센서(렌즈 감지, EMF)에 의존해야 하는 환경입니다.

#### 페인포인트

- Wi-Fi가 없는 탈의실에서 네트워크 스캔이 무의미
- 매일 이용하는 곳이라 매번 정밀 스캔은 비현실적
- 다른 이용자가 보는 앞에서 스마트폰으로 스캔하는 것이 어색

#### 니즈

- Wi-Fi 없이도 동작하는 탐지 기능 (렌즈 찾기, EMF)
- 빠르고 눈에 띄지 않는 스캔 방법
- 첫 방문 시 Full Scan, 이후 주기적 Quick Check

#### 앱 사용 시나리오

1. 첫 방문 시 탈의실에서 "렌즈 찾기" 모드를 실행합니다
2. 플래시를 켜고 사물함 위, 거울 주변, 천장을 비춥니다
3. 의심 포인트가 없으면 안심합니다
4. 이후에는 주 1회 EMF 스캔으로 변화가 없는지 확인합니다

| **기술 수준** | 하~중 (스마트폰 기본 기능 사용) |
| **연소득** | 5,400만원 |
| **스마트폰** | Galaxy A54 |

#### 상황

중학생 딸이 수학여행을 갑니다. 숙소에서 몰래카메라가 걱정되지만, 딸에게 불안감을 주기는 싫습니다. 앱을 깔아서 딸에게 알려주거나, 직접 점검해주고 싶습니다.

#### 페인포인트

- 기술에 익숙하지 않아 복잡한 앱 사용이 어려움
- 자녀에게 "몰래카메라"라는 주제를 어떻게 전달할지 고민
- 오탐이 나오면 대처 방법을 모름

#### 니즈

- 극도로 쉬운 UI (큰 글씨, 명확한 색상)
- 자녀에게 건네줄 수 있는 간단한 사용 가이드
- "어떻게 하세요" 수준의 단계별 안내

#### 앱 사용 시나리오

1. 딸의 수학여행 전날 앱을 설치합니다
2. 온보딩을 따라 사용법을 익힙니다
3. 딸에게 "숙소 들어가면 이 버튼 한번 눌러봐"라고 알려줍니다
4. 딸이 결과 스크린샷을 카카오톡으로 보내줍니다

### 3.6 페르소나 6: 윤재혁 (펜션 사업자)

| 항목 | 내용 |
|------|------|
| **이름** | 윤재혁 |
| **나이** | 55세 |
| **직업** | 펜션 운영 사업자 |
| **거주지** | 강원 속초시 |
| **기술 수준** | 중 |
| **연소득** | 7,200만원 |
| **스마트폰** | Galaxy S24+ |

#### 상황

자신이 운영하는 펜션에 이전 투숙객이 몰래카메라를 설치했다는 의혹이 인터넷 커뮤니티에 올라왔습니다. 사실이 아니지만, 투숙객 신뢰를 회복하기 위해 정기 점검을 하고 결과를 공개하고 싶습니다.

#### 페인포인트

- 전문 점검 업체를 매번 부르면 비용이 큼 (객실 10개)
- 점검 결과를 투숙객에게 신뢰할 수 있게 공유하고 싶음
- 매주 정기 점검 루틴이 필요

#### 니즈

- 객실별 스캔 리포트 관리 (날짜별, 객실별)
- PDF 리포트 내보내기 (투숙객 공유용)
- 정기 점검 리마인더

#### 앱 사용 시나리오

1. 매주 월요일 오전, 각 객실을 돌며 Full Scan을 실행합니다
2. 리포트를 PDF로 저장합니다
3. 펜션 홈페이지에 "주간 안전 점검 완료" 배지와 함께 게시합니다
4. 투숙객 문의 시 해당 날짜 리포트를 공유합니다

## 4. 사용자 여정 맵 (Customer Journey Map)

### 4.1 단계별 여정

```
인지 → 탐색 → 설치 → 첫 사용 → 반복 사용 → 추천/이탈
```

### 4.2 상세 여정 맵 (페르소나 1: 김서연 기준)

| 단계 | 행동 | 감정 | 접점 | 기회 |
|------|------|------|------|------|
| **인지** | SNS에서 몰래카메라 피해 사례 접함 | 불안, 걱정 | Instagram, 뉴스 | 관련 키워드 SEO, 소셜 미디어 콘텐츠 |
| | "탐지 앱이 있다던데" 검색 | 호기심 | Google, 네이버 | ASO 최적화 |
| **탐색** | Play Store에서 "몰래카메라 탐지" 검색 | 비교, 탐색 | Play Store | 스크린샷, 평점, 설명 최적화 |
| | 리뷰 확인 (오탐 관련 부정 리뷰 다수) | 의심, 망설임 | Play Store 리뷰 | 솔직한 한계 고지로 신뢰 확보 |
| | SearCam 발견 "교차 검증으로 오탐 5%" | 관심 | Play Store 설명 | 차별점 강조 |
| **설치** | SearCam 다운로드 | 기대 | Play Store | 앱 크기 최소화 (50MB 이하) |
| | 온보딩 3장 확인 | 이해 | 앱 내 온보딩 | 30초 원칙 + 한계 솔직 고지 |
| | 권한 허용 (Wi-Fi, 위치, 카메라) | 약간의 경계 | 시스템 다이얼로그 | 각 권한의 이유를 명확히 설명 |
| **첫 사용** | 홈 화면 → Quick Scan 탭 | 긴장, 기대 | 홈 화면 | 큰 CTA 버튼으로 직관적 시작 |
| | 30초 스캔 진행 | 약간의 불안 | 스캔 진행 화면 | 진행률 + 발견 기기 수 실시간 표시 |
| | "안전 12/100" 결과 확인 | 안도 | 결과 화면 | 근거 기반 결과 (기기 목록 + 한계 고지) |
| | Full Scan 권유 배너 확인 | 고민 | 결과 화면 | 강요 아닌 부드러운 권유 |
| **반복 사용** | 다음 여행에서 다시 사용 | 습관화 | 앱 | 알림 리마인더 (숙소 위치 감지 시) |
| | 렌즈 찾기 모드 시도 | 호기심 | 렌즈 찾기 화면 | 사용 가이드 오버레이 |
| | 리포트 저장 | 안심 | 리포트 화면 | PDF 내보내기 (프리미엄) |
| **추천** | 친구에게 카카오톡으로 공유 | 만족 | 카카오톡 | 공유 딥링크, 레퍼럴 보상 |
| | Play Store 리뷰 작성 | 호의적 | Play Store | 리뷰 요청 타이밍 (3회 사용 후) |

### 4.3 이탈 지점과 대응

| 이탈 지점 | 원인 | 대응 방안 |
|----------|------|----------|
| 온보딩 | 너무 길거나 지루 | 3장 이하, 건너뛰기 제공 |
| 권한 요청 | 위치 권한에 대한 거부감 | "Wi-Fi 스캔에 필수" 설명 + "위치 저장 안 함" 명시 |
| 첫 스캔 결과 | 오탐으로 불안감 증가 | 근거 설명 + "확인 필요" 표현 (공포 방지) |

| 유료 전환 | 핵심 기능이 유료 | 핵심은 완전 무료, 부가 기능만 유료 |

## 5. 사용성 테스트 계획

### 5.1 테스트 개요

| 항목 | 내용 |
|------|------|
| 방법 | 과업 기반 사용성 테스트 (Task-based Usability Testing) |
| 인원 | 8명 (페르소나별 1~2명 배정) |
| 환경 | 실제 숙박 시설 또는 모의 환경 (Wi-Fi AP + 테스트 기기 설치) |
| 도구 | 프로토타입 (Figma Interactive) + 실제 빌드 |
| 기록 | 화면 녹화 + 음성 + 관찰자 노트 |
| 시간 | 1인당 60분 (과업 30분 + 인터뷰 30분) |

### 5.2 테스트 시나리오

| # | 과업 | 성공 기준 | 측정 지표 |
|---|------|----------|----------|
| T1 | 앱을 열고 첫 스캔을 완료하세요 | 온보딩 → 권한 → Quick Scan → 결과 도달 | 완료 시간, 에러 횟수, 도움 요청 횟수 |
| T2 | 스캔 결과를 확인하고 의미를 설명해주세요 | 위험도 등급과 발견 기기를 정확히 이해 | 정확한 설명 비율, 이해도 (1~5점) |
| T3 | 렌즈 찾기 모드를 실행하고 방을 스캔하세요 | 렌즈 찾기 진입 → 플래시 ON → 스캔 수행 | 진입 시간, 스캔 커버리지, 자연스러움 |
| T4 | 이전 스캔 결과 리포트를 찾아보세요 | 리포트 탭 → 해당 리포트 찾기 | 완료 시간, 클릭 수 |
| T5 | 체크리스트로 숙소를 육안 점검하세요 | 체크리스트 선택 → 5개 항목 체크 | 완료 시간, 체크리스트 이해도 |
| T6 | 스캔 결과에서 "의심" 기기의 상세 정보를 확인하세요 | 기기 탭 → 상세 화면 → MAC/포트 정보 확인 | 발견 시간, 정보 이해도 |
| T7 | 112에 신고하는 방법을 찾으세요 | 결과 화면 → 112 신고 버튼 발견 | 발견 시간 |

### 5.3 측정 지표

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 과업 성공률 | 도움 없이 완료한 비율 | 85%+ |
| 과업 완료 시간 | 과업 시작~완료까지 시간 | Quick Scan 2분 이내 |
| 에러율 | 잘못된 탭, 혼란 횟수 | 과업당 1회 이하 |
| SUS (System Usability Scale) | 10문항 설문 | 70점+ (Good) |
| NPS (Net Promoter Score) | 추천 의향 (0~10) | 40+ |
| 이해도 | 결과 해석 정확도 (1~5) | 4.0+ |

### 5.4 테스트 후 개선 우선순위 프레임워크

| 심각도 | 기준 | 대응 |
|--------|------|------|

| Major | 과업 완료 가능하나 큰 불편 | MVP 출시 전 수정 권장 |

## 6. 경험 원칙 (Design Principles from User Research)

### 6.1 핵심 경험 원칙

리서치 결과를 바탕으로 도출된 SearCam만의 경험 원칙입니다.

#### 원칙 1: 안심은 속도에서 온다

> 30초 안에 첫 번째 답을 줘라. 정확한 답보다 빠른 답이 우선이다.

- 근거: 사용자의 불안은 "모르는 상태"에서 발생. 빠른 1차 결과가 불안을 줄임
- 구현: Quick Scan 30초, 결과 즉시 표시, 이후 정밀 스캔 선택권 부여

#### 원칙 2: 정직이 신뢰를 만든다

> 과장하지 마라. 못 하는 것을 솔직히 말해라.

- 근거: 기존 앱의 최대 불만 = "다 위험이라고 뜨는데 정작 아무것도 못 찾음"
- 구현: 한계 고지 상시 표시, 오탐 가능성 명시, "전문 장비 대체 불가" 안내

#### 원칙 3: 근거 없는 결과는 없다

> "위험"이라고만 하지 마라. 왜 위험한지 증거를 보여줘라.

- 근거: 사용자는 "위험"이라는 단어에 공포를 느낌. 근거가 있으면 대처 가능
- 구현: 레이어별 점수 + 구체적 근거 (MAC 주소, 포트, 자기장 수치 등)

#### 원칙 4: 기술을 숨기고 안심을 보여줘라

> 사용자는 ARP, mDNS, retroreflection을 알 필요 없다.

- 근거: 기술 수준이 낮은 사용자(페르소나 3, 5)도 쉽게 사용해야 함
- 구현: 기본 UI는 색상 + 한국어 안내. 기술 상세는 "더 보기" 뒤에 숨김

#### 원칙 5: 한 번 쓰고 버리는 앱이 아니다

> 반복 사용을 유도하되, 강요하지 마라.

- 근거: 여행, 출장, 피트니스 등 반복 사용 맥락이 존재
- 구현: 마지막 스캔 결과 카드, 리포트 히스토리, 리마인더 (선택적)

#### 원칙 6: 최악의 순간에 가장 단순해야 한다

> "의심" 결과가 나왔을 때, 다음 행동이 명확해야 한다.

- 근거: 위험 상황에서 사용자의 인지 능력은 저하됨
- 구현: 위험 등급별 명확한 행동 안내 (정밀 확인 / 관리자 문의 / 112 신고)
