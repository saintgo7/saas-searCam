## 1. 리스크 평가 방법론

### 1.1 평가 프레임워크

본 문서는 **확률 x 영향도** 매트릭스를 사용하여 SearCam 프로젝트의 리스크를 체계적으로 식별, 평가, 관리한다.

### 1.2 확률 등급

| 등급 | 수치 | 정의 | 기준 |
|------|------|------|------|
| 극히 낮음 | 1 | 발생 가능성 거의 없음 | 5% 미만 |
| 낮음 | 2 | 발생 가능성 낮음 | 5~20% |
| 보통 | 3 | 발생할 수 있음 | 20~50% |
| 높음 | 4 | 발생 가능성 높음 | 50~80% |
| 매우 높음 | 5 | 거의 확실히 발생 | 80% 이상 |

### 1.3 영향도 등급

| 등급 | 수치 | 정의 | 기준 |
|------|------|------|------|
| 무시 | 1 | 영향 거의 없음 | 일정/비용 1% 미만 변동 |
| 경미 | 2 | 소규모 영향 | 일정 1~2주 지연 또는 비용 5% 증가 |
| 보통 | 3 | 중간 수준 영향 | 일정 2~4주 지연 또는 비용 10% 증가 |
| 심각 | 4 | 주요 기능/일정 영향 | 일정 1~2개월 지연 또는 핵심 기능 제한 |
| 치명적 | 5 | 프로젝트 중단 수준 | 프로젝트 목표 달성 불가 또는 법적 문제 |

### 1.4 리스크 수준 분류

| 점수 범위 | 리스크 수준 | 색상 | 대응 원칙 |
|-----------|-----------|------|----------|
| 1~4 | 낮음 (Low) | 초록 | 모니터링, 별도 대응 불필요 |
| 5~9 | 보통 (Medium) | 노랑 | 대응 계획 수립, 정기 검토 |
| 10~15 | 높음 (High) | 주황 | 즉시 대응 계획 실행, 주간 검토 |
| 16~25 | 심각 (Critical) | 빨강 | 최우선 대응, 에스컬레이션 |

## 8. 에스컬레이션 프로세스

```
리스크 발생/변동
       │
       ▼
[리스크 수준 재평가]
       │
       ├── 낮음/보통 → 담당자가 대응 계획 실행 → 정기 검토에서 보고
       │
       ├── 높음 → PM에게 즉시 보고 → 24시간 내 대응 계획 수립
       │          → 주간 리뷰에서 진행 상황 추적
       │
       └── 심각 → PM + 전체 팀 긴급 회의 (24시간 내)
                  → 대응 계획 즉시 실행
                  → 매일 진행 상황 보고
                  → 필요 시 외부 자문 (법률, 보안)
```

### 에스컬레이션 조건

| 조건 | 에스컬레이션 대상 |
|------|----------------|
| 리스크 점수 상승 (보통 → 높음) | PM |
| 리스크 점수 상승 (높음 → 심각) | PM + 전체 팀 |
| 대응 마감일 초과 | PM |
| 신규 리스크 발견 (높음 이상) | PM + 전체 팀 |
| 법적 이슈 발생 | PM + 법무 |
| 사용자 안전 관련 이슈 | PM + 전체 팀 + 법무 |
