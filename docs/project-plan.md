## 1. 프로젝트 개요

### 1.1 프로젝트명
**SearCam** (Search + Camera)

### 1.2 핵심 목표
> 일반인이 숙소·화장실·탈의실에서 **30초 안에** 스마트폰만으로 몰래카메라를 점검할 수 있는 앱

### 1.3 현실 인식

전문 탐지 장비(RF 스캐너, NLJD 등)는 전용 하드웨어와 Monitor Mode Wi-Fi로 높은 정확도를 달성합니다. **스마트폰은 이 방식을 그대로 재현할 수 없습니다.**

**따라서 본 계획서는 "스마트폰에서 실제로 가능한 것"만으로 설계하고, 특히 플래시 Retroreflection 렌즈 감지를 핵심 차별점으로 채택합니다.**

### 1.4 스마트폰에서 실제로 되는 것 / 안 되는 것

| 탐지 방식 | 가능 여부 | 현실적 정확도 | 설명 |
|----------|----------|-------------|------|
| **네트워크 기기 스캔** (ARP, mDNS, 포트) | Android: O, iOS: 제한적 | 80~90% (Wi-Fi 카메라 한정) | 같은 네트워크에 연결된 카메라만 |
| **MAC OUI 매칭** | Android: O, iOS: 제한적 | 보조 수단 | 카메라 제조사 MAC 식별 |
| **자기장(EMF) 감지** | 양쪽 모두 O | 40~60% (단독) | 오탐 높음, 교차검증 필수 |
| **IR 적외선 감지** | 양쪽 모두 O | 70~80% (암실 조건) | 어두운 환경에서만 유효 |
| **렌즈 반사 감지** (플래시) | 양쪽 모두 O | 60~70% | 거울/금속도 반사 → 오탐 |
| **ToF/LiDAR 렌즈 감지** | iPhone Pro만 | 85~89% (LAPD 논문) | 사용자 ~10%만 해당 |
| Wi-Fi CSI 분석 | **불가** | - | root/특수 펌웨어 필요 |
| Wi-Fi 트래픽 DPI | **불가** | - | Monitor Mode 필요 |
| EM 방사 분석 | **불가** | - | 외부 하드웨어 필요 |
| 열화상 분석 | **불가** | - | 외부 열화상 카메라 필요 |

## 2. 탐지 가능한 카메라 유형

사용자가 오해하지 않도록, 탐지 가능/불가능 범위를 명확히 합니다.

### 2.1 탐지 가능

| 카메라 유형 | 탐지 방식 | 탐지율 |
|-----------|----------|--------|
| **Wi-Fi 연결 IP 카메라** (같은 네트워크) | Wi-Fi 스캔 + OUI + 포트 | **80~85%** |
| **렌즈 노출 카메라** (전원 무관) | 플래시 Retroreflection | **70~80%** |
| **IR LED 탑재 카메라** (야간 촬영형) | IR 감지 (암실) | **70~80%** |
| **전원 연결 소형 카메라** | 자기장 감지 (근접) | **50~60%** |
| **Wi-Fi AP 내장 카메라** (자체 핫스팟) | Wi-Fi 스캔 (SSID) | **70~80%** |

### 2.2 탐지 어려움

| 카메라 유형 | 이유 | 대응 |
|-----------|------|------|
| **LTE/5G 카메라** (SIM 내장) | Wi-Fi 미사용, 독립 네트워크 | EMF + IR로만 보조 탐지 |
| **SD카드 녹화 카메라** (오프라인) | 무선 신호 없음 | EMF (전원 있을 때만) + 체크리스트 |
| **배터리 + SD 초소형** (완전 독립) | 전자기 신호 극소 | 육안 체크리스트가 유일 |
| **전원 OFF 카메라** | 모든 신호 없음 | 탐지 불가 → 솔직히 고지 |

### 2.3 앱 내 솔직한 안내 문구
```
SearCam은 Wi-Fi 연결 카메라, IR 야간 카메라, 전원이 켜진 소형 카메라를
탐지하는 1차 스크리닝 도구입니다.

탐지되지 않았다고 100% 안전을 보장하지 않습니다.
의심 시 전문 탐지 업체 또는 경찰(112)에 신고해주세요.

전문 탐지 장비(RF 스캐너, NLJD 등)를 대체하지 않습니다.
```

## 4. 기술 아키텍처

### 4.1 탐지 레이어 설계

**스마트폰에서 실제 가능한 범위로 설계**한 3-Layer 탐지 파이프라인입니다.

```
┌─────────────────────────────────────────────┐
│           SearCam 탐지 파이프라인            │
├─────────────────────────────────────────────┤
│                                              │
│  Layer 1: 네트워크 스캔 (가중치 50%)         │
│  ┌─────────────────────────────────────┐    │
│  │ ARP 테이블 → 네트워크 기기 목록     │    │
│  │ mDNS/SSDP → 서비스 광고 기기 발견   │    │
│  │ MAC OUI → 카메라 제조사 매칭        │    │
│  │ 포트 스캔 → RTSP/HTTP 스트리밍 확인 │    │
│  │ 호스트명 → "cam", "ipcam" 패턴 매칭 │    │
│  └─────────────────────────────────────┘    │
│                                              │
│  Layer 2: 렌즈 감지 (가중치 35%)              │
│  ┌─────────────────────────────────────┐    │
│  │ Stage A: 플래시 Retroreflection     │    │
│  │   후면 카메라+플래시 → 렌즈 역반사  │    │
│  │   AI 패턴 분석 → 렌즈 vs 일반 반사  │    │
│  │   밝은 환경 OK, 전원 OFF도 탐지     │    │
│  │ Stage B: IR LED 감지                │    │
│  │   전면 카메라 → IR 발광 감지 (암실)  │    │
│  └─────────────────────────────────────┘    │
│                                              │
│  Layer 3: 자기장 감지 (가중치 15%)           │
│  ┌─────────────────────────────────────┐    │
│  │ 캘리브레이션 → 기준 자기장 측정     │    │
│  │ 실시간 측정 → 20Hz 3축 데이터      │    │
│  │ 노이즈 필터 → 급변 신호 제거       │    │
│  │ 임계값 판정 → 변화량 기반 등급     │    │
│  └─────────────────────────────────────┘    │
│                                              │
│  ┌─────────────────────────────────────┐    │
│  │         교차 검증 엔진               │    │
│  │  3개 레이어 결과 종합 → 위험도 산출 │    │
│  │  복수 양성 시 가중 보정              │    │
│  │  근거 기반 결과 설명                 │    │
│  └─────────────────────────────────────┘    │
│                                              │
└─────────────────────────────────────────────┘
```

### 4.2 Layer 1: 네트워크 스캔 (상세)

**스마트폰에서 실제 가능한 네트워크 분석만 포함합니다.**

#### 4.2.1 스캔 순서

```
Step 1: 네트워크 연결 확인
  └─ Wi-Fi 미연결 → Layer 1 스킵, Layer 2+3만 실행

Step 2: ARP 테이블 조회 (Android)
  └─ /proc/net/arp 읽기 → 같은 네트워크 기기 IP+MAC 목록
  └─ iOS: 직접 조회 불가 → mDNS/Bonjour로 대체

Step 3: mDNS/SSDP 서비스 탐색
  └─ _rtsp._tcp → RTSP 스트리밍 서비스 (카메라 강력 의심)
  └─ _http._tcp → 웹 인터페이스 (카메라 관리 페이지)
  └─ UPnP SSDP M-SEARCH → 네트워크 기기 발견

Step 4: MAC OUI 매칭
  └─ MAC 앞 3바이트 → IEEE OUI 데이터베이스 조회
  └─ 카메라 제조사 매칭 (Hikvision, Dahua, Wyze 등 300+사)
  └─ risk_weight 부여 (ip_camera: 0.9, smart_camera: 0.7)

Step 5: 포트 스캔 (의심 기기만)
  └─ RTSP (554) → 영상 스트리밍 프로토콜
  └─ HTTP (80, 8080, 8888) → 카메라 웹 인터페이스
  └─ ONVIF (3702) → IP 카메라 표준 프로토콜
  └─ RTMP (1935) → 라이브 스트리밍

Step 6: 호스트명/서비스명 패턴 매칭
  └─ "cam", "ipcam", "dvr", "nvr", "stream", "video" 등
```

#### 4.2.2 위험도 산출 (Layer 1)

```
기기별 위험도 점수 (0~100):

  카메라 제조사 MAC 일치:        +40점
  RTSP 포트(554) 개방:           +30점
  HTTP 스트리밍 포트 개방:       +15점
  ONVIF 포트(3702) 개방:         +20점
  호스트명에 카메라 키워드:      +10점
  mDNS _rtsp 서비스 광고:        +25점

  최종 = min(100, 합산)
```

#### 4.2.4 Android vs iOS 차이

| 기능 | Android | iOS |
|------|---------|-----|
| ARP 테이블 | `/proc/net/arp` 직접 읽기 | 접근 불가 |
| MAC 주소 조회 | 가능 | iOS 14+ 랜덤 MAC |
| mDNS/Bonjour | NsdManager | NWBrowser (Network.framework) |

| Wi-Fi 스캔 (SSID 목록) | WifiManager | NEHotspotHelper (특별 권한) |
| **현실적 대응** | **풀 기능** | **mDNS + 포트 스캔 중심** |

### 4.3 Layer 2: 렌즈 감지 (Retroreflection + IR) (상세)

> **v3.1 핵심 변경**: 기존 IR 감지를 **플래시 Retroreflection 렌즈 감지**로 대폭 강화.
> 사용자가 플래시를 켜고 촬영하면, AI가 카메라 렌즈 특유의 반사 패턴을 온디바이스에서 실시간 분석합니다.

#### 4.3.1 원리: Retroreflection이란?

카메라 렌즈는 물리적 구조(볼록 렌즈 + 이미지 센서) 때문에, 빛이 들어온 방향으로 되돌아가는 **역반사(retroreflection)** 특성이 있습니다. 이것은 거울이나 금속의 반사와 물리적으로 다릅니다.

```
[카메라 렌즈 retroreflection]

  플래시 빛 ───→  ◉ 렌즈  ───→ 빛이 되돌아옴 (역반사)
                  │        │
                  │ 볼록렌즈 │
                  │ + 센서  │
                  └────────┘

  특징:
  • 매우 밝고 작은 점 (1~5 pixel)
  • 플래시 방향으로만 반사 (관찰자에게 직접 돌아옴)
  • 색상: 순백색 또는 약간 붉은빛
  • 위치 고정 (카메라가 고정되어 있으므로)
  • 주변 밝기와 무관하게 뚜렷함

[일반 반사 (거울/금속/유리)]

  플래시 빛 ───→  ▭ 평면  ───→ 빛이 넓게 퍼짐 (정반사/난반사)

  특징:
  • 넓은 면적 반사 (수십~수백 pixel)
  • 각도에 따라 위치/밝기 변화
  • 색상: 원래 표면 색상 반영
  • 스마트폰 이동 시 반사 위치도 이동
```

**학술적 근거**: ACM SenSys 2021 LAPD 논문에서 ToF 레이저로 동일 원리 활용, 88.9% 탐지율 기록. SearCam은 **일반 플래시**로 동일 원리를 적용하되, AI 패턴 분석으로 보완.

#### 4.3.2 2단계 탐지 구조

```
┌─────────────────────────────────────────────────┐
│         Layer 2: 렌즈 감지 (2단계)               │
│                                                   │
│  ┌─── Stage A: 플래시 Retroreflection ────────┐  │
│  │                                              │  │
│  │  후면 카메라 + 플래시 ON                     │  │
│  │  사용자가 방을 천천히 촬영                   │  │
│  │  AI가 렌즈 반사 패턴 실시간 감지             │  │
│  │  → 밝은 환경에서도 작동 (핵심 장점)         │  │
│  │                                              │  │
│  │  탐지 가능: 모든 카메라 렌즈 (전원 무관)    │  │
│  │  탐지율: 70~80% (단독)                      │  │
│  └──────────────────────────────────────────────┘  │
│                                                   │
│  ┌─── Stage B: IR LED 감지 ───────────────────┐  │
│  │                                              │  │
│  │  전면 카메라 (IR 필터 약한 카메라)           │  │
│  │  어두운 환경에서 IR LED 발광 감지            │  │
│  │  → 야간 촬영 카메라 전용                    │  │
│  │                                              │  │
│  │  탐지 가능: IR LED 탑재 카메라만             │  │
│  │  탐지율: 70~80% (암실 조건)                 │  │
│  └──────────────────────────────────────────────┘  │
│                                                   │
│  Stage A + B 교차 = Layer 2 최종 점수             │
└─────────────────────────────────────────────────┘
```

#### 4.3.3 Stage A: 플래시 Retroreflection 감지 알고리즘

```
[사용자 가이드]
"플래시가 켜진 상태로 벽면, 선반, 천장을
천천히 비춰주세요 (속도: 초당 30cm 이하)"

[AI 분석 파이프라인 — 온디바이스 실시간]

입력: 후면 카메라 프레임 (30fps, 플래시 ON)

1. 프레임 전처리
   └─ 해상도 축소 (720p → 분석용)
   └─ 그레이스케일 변환

2. 고휘도 포인트 추출
   └─ 적응형 임계값 (주변 밝기의 상위 0.1%)
   └─ 플래시 ON이므로 전체적으로 밝음 → 상대적 임계값 사용
   └─ 포인트 크기 필터: 1~10 pixel (렌즈 반사는 매우 작음)

3. Retroreflection 특성 검사 (핵심 — 오탐 감소)
   ┌────────────────────────────────────────────────┐
   │ 검사 항목              │ 렌즈         │ 일반 반사 │
   ├────────────────────────┼──────────────┼──────────┤
   │ 포인트 크기            │ 1~10 px      │ 10+ px   │
   │ 형태                   │ 원형 (원형도>0.8) │ 불규칙   │
   │ 밝기 vs 주변 대비      │ 극도로 높음  │ 중간     │
   │ 위치 안정성 (5프레임)  │ 고정         │ 흔들림   │
   │ 색상 (RGB 분석)        │ 순백~붉은빛  │ 다양     │
   │ 스마트폰 이동 시       │ 위치 고정    │ 같이 이동│
   └────────────────────────┴──────────────┴──────────┘

4. 시간축 분석 (2초 윈도우)
   └─ 2초간 동일 위치에 지속: 의심 포인트로 승격
   └─ 깜빡이는 포인트: LED 가능성 → 제외
   └─ 이동하는 포인트: 반사 잔상 → 제외

5. 동적 검증 (자동)
   └─ 플래시 순간 OFF (0.2초) → 포인트 소실 확인
   └─ 소실되면: 플래시 의존 반사 = 렌즈 가능성 ↑
   └─ 유지되면: 자체 발광 (LED 등) = 렌즈 아닐 가능성

6. 의심 포인트 출력
   └─ 화면에 빨간 원 + 펄스 애니메이션
   └─ 진동 알림
   └─ "이 방향에 렌즈가 의심됩니다. 가까이 확인해주세요"
```

#### 4.3.4 Stage A: 위험도 채점

```
포인트별 위험도 (0~100):

  크기 1~5px + 원형도 > 0.8:           +30점
  2초 이상 위치 고정:                   +20점
  플래시 OFF 시 소실:                   +25점
  주변 대비 밝기 상위 0.05%:           +15점
  RGB 순백 또는 붉은빛:                +10점

  감점:
  크기 > 15px (넓은 면적 반사):        -30점
  스마트폰 이동 시 같이 이동:          -40점 (거울/유리)
  불규칙 형태:                         -20점

  최종 = max(0, 합산)
```

#### 4.3.5 Stage B: IR LED 감지 (기존 유지)

```
조건: 어두운 환경 + 전면 카메라

1. 환경 밝기 체크
   └─ 밝으면 → "방을 어둡게 해주세요" 안내

2. 전면 카메라 프레임 분석 (30fps)
   └─ 고휘도 포인트 추출
   └─ IR LED 특성: 보라색/흰색, 지속적 발광, 위치 고정

3. 시간축 필터 (3초 윈도우)
   └─ 3초+ 지속 = 의심 (카메라 IR LED)
   └─ 깜빡임 = 일반 LED → 제외
   └─ 이동 = 반사 → 제외

4. 의심 포인트 출력
   └─ 화면에 보라색 원 하이라이트
```

#### 4.3.6 Stage A vs Stage B 비교

| 항목 | Stage A (Retroreflection) | Stage B (IR) |
|------|--------------------------|-------------|
| 사용 카메라 | 후면 + 플래시 | 전면 (IR 필터 약한) |
| 조명 조건 | **밝은 환경 OK** | 어두운 환경만 |
| 탐지 대상 | **모든 카메라 렌즈** (전원 무관) | IR LED 탑재 카메라만 |
| 원리 | 플래시 역반사 | IR 발광 감지 |
| 장점 | 전원 꺼진 카메라도 탐지 가능 | 야간 카메라 확실한 탐지 |
| 단점 | 거울/유리 오탐 가능 | 밝은 곳 사용 불가 |

> **Stage A가 핵심 혁신**: 기존 앱들은 IR 감지만 제공.
> SearCam은 **플래시 retroreflection**으로 밝은 환경 + 전원 OFF 카메라까지 탐지 시도.

#### 4.3.7 렌즈 감지가 안 되는 경우 (솔직한 한계)

| 상황 | 이유 | 대응 |
|------|------|------|
| 렌즈가 덮개로 가려진 경우 | 반사 자체가 안 됨 | 체크리스트로 육안 확인 안내 |
| 핀홀 카메라 (0.5mm 렌즈) | 반사 면적 극소 → 감지 어려움 | EMF 레이어에 의존 |
| 스마트폰 각도가 안 맞을 때 | 역반사는 빛 방향 의존 | "다양한 각도로 비춰주세요" 안내 |
| 렌즈 반사 코팅된 카메라 | 고급 카메라는 반사 억제 코팅 | 탐지율 하락 가능 → 한계 명시 |

### 4.4 Layer 3: 자기장(EMF) 감지 (상세)

#### 4.4.1 원리
- 모든 전자기기는 동작 시 자기장 발생
- 카메라: 렌즈 모터, 회로, 무선 송신기에서 자기장 발생
- 스마트폰 3축 자력계로 주변 자기장 변화 측정

#### 4.4.2 측정 알고리즘

```
Phase 0: 캘리브레이션 (3초)
  ┌─────────────────────────────────────────┐
  │ 사용자: "스마트폰을 공중에 들고 대기"    │
  │                                          │
  │ 3초간 자기장 측정 (60 samples at 20Hz)   │
  │ baseline = mean(sqrt(x² + y² + z²))      │
  │ noise_floor = std_dev(samples) × 2       │
  └─────────────────────────────────────────┘

Phase 1: 실시간 스캔
  ┌─────────────────────────────────────────┐
  │ 사용자: 벽/가구/물체에 스마트폰을 이동   │
  │                                          │
  │ 50ms 간격 측정 (20Hz)                    │
  │ magnitude = sqrt(x² + y² + z²)           │
  │ delta = |magnitude - baseline|            │
  │                                          │
  │ 노이즈 필터:                             │
  │   |delta| < noise_floor → 무시           │
  │   0.3초 내 급변 > 50μT → 자체 간섭 무시  │
  │                                          │
  │ 이동 평균 (window = 10) 적용             │
  └─────────────────────────────────────────┘

Phase 2: 판정
  ┌──────────────┬──────────┬────────┐
  │ 변화량 (μT)  │ 등급     │ 점수   │
  ├──────────────┼──────────┼────────┤
  │ < 5          │ 정상     │ 0      │
  │ 5 ~ 15       │ 관심     │ 20     │
  │ 15 ~ 30      │ 주의     │ 50     │
  │ 30 ~ 50      │ 의심     │ 75     │
  │ > 50         │ 강한 의심│ 95     │
  └──────────────┴──────────┴────────┘
```

#### 4.4.3 오탐 원인과 대응

| 오탐 원인 | 자기장 특성 | 필터 방법 |
|----------|-----------|----------|
| 철근/금속 구조물 | 지속적, 위치 고정 | 이동 시 급감하면 구조물 |
| 가전 (TV, 냉장고) | 강하고 넓은 범위 | 범위 큰 신호는 가전 가능성 고지 |
| 전선/콘센트 | 일정한 자기장 | 위치가 콘센트 근처면 안내 |
| 스마트폰 자체 | 급격한 변동 | 0.3초 내 급변 필터 |
| 자석 (가방 걸쇠 등) | 매우 강하고 고정 | > 100μT는 자석 가능성 안내 |

#### 4.4.4 감도 설정

| 모드 | 임계값 | 용도 | 오탐 수준 |
|------|--------|------|----------|
| 민감 | 3 μT | 초소형 카메라 탐지 시도 | 높음 (30%+) |
| 보통 (기본) | 8 μT | 일반 탐지 | 중간 (10~15%) |
| 안정 | 20 μT | 큰 기기만, 오탐 최소 | 낮음 (5%) |

### 4.5 교차 검증 엔진

**핵심 차별점: 단일 센서 결과가 아닌 복수 레이어 교차 검증**

#### 4.5.1 가중치 전략

```
기본 가중치:
  Wi-Fi 스캔:      W1 = 0.50
  렌즈 감지 (A+B): W2 = 0.35
  자기장:          W3 = 0.15

동적 조정:
  Wi-Fi 미연결 시:            W1=0, W2=0.75, W3=0.25
  IR 불가 (밝은 환경):        W1=0.55, W2=0.30 (Stage A만), W3=0.15
  Wi-Fi 미연결 + 밝은 환경:   W1=0, W2=0.80 (Stage A만), W3=0.20
  Wi-Fi 없는 화장실 (암실):   W1=0, W2=0.75, W3=0.25
```

#### 4.5.2 교차 검증 매트릭스

```
종합 위험도 = (W1 × L1_score + W2 × L2_score + W3 × L3_score) × 보정계수

보정 계수:
  ┌──────────────────────────────┬──────────┐
  │ 조건                         │ 보정     │
  ├──────────────────────────────┼──────────┤
  │ 1개 레이어만 양성            │ × 0.7    │
  │ 2개 레이어 양성              │ × 1.2    │
  │ 3개 모두 양성                │ × 1.5    │
  │ Wi-Fi + IR 동시 양성 (같은 방향) │ × 1.4 │
  │ Wi-Fi만 양성 (나머지 음성)   │ × 0.9   │
  │ 자기장만 양성 (나머지 음성)  │ × 0.5   │
  └──────────────────────────────┴──────────┘

  자기장 단독 양성의 보정이 0.5인 이유:
  → 자기장은 오탐률이 40~60%로 높음
  → 단독 양성은 가전/금속일 확률이 더 높음
  → "주의" 등급으로 표시하되 "카메라 확정"은 하지 않음
```

#### 4.5.3 위험도 등급

| 점수 | 등급 | 색상 | 사용자 안내 |
|------|------|------|------------|
| 0~19 | **안전** | 초록 | "이상이 감지되지 않았습니다" |
| 20~39 | **관심** | 연두 | "약한 신호가 있으나 일반 기기일 가능성이 높습니다" |
| 40~59 | **주의** | 노랑 | "의심 신호가 있습니다. 해당 위치를 직접 확인해주세요" |
| 60~79 | **위험** | 주황 | "카메라 가능성이 있습니다. 정밀 확인을 권장합니다" |
| 80~100 | **매우 위험** | 빨강 | "카메라가 강하게 의심됩니다. 관리자 또는 경찰(112)에 신고해주세요" |

### 5.5 체크리스트 (육안 점검 가이드)

센서로 탐지 불가능한 카메라(오프라인, 전원 OFF)도 있으므로, **육안 점검 가이드를 필수로 제공**합니다.

**숙소 체크리스트 (15항목)**:
```
■ 시선 방향 (침대/소파를 향한 곳)
  □ 연기 감지기 → 렌즈 구멍 확인
  □ 시계 → 디지털 시계 뒷면/숫자 주변
  □ TV 베젤 → 작은 구멍
  □ 에어컨 리모컨 → IR 센서 부위
  □ USB 충전기/어댑터 → 충전 포트 외 구멍

■ 벽면/천장
  □ 액자 → 뒷면 작은 구멍
  □ 나사/못 → 핀홀 카메라 위장
  □ 환풍구 → 격자 사이
  □ 스프링클러 → 렌즈 형태

■ 욕실
  □ 거울 → 양면 거울 테스트 (손톱 대기, 간격 0 = 양면)
  □ 샤워기 홀더 → 각도 확인
  □ 비누/샴푸통 → 이상한 구멍
  □ 휴지 거치대 → 나사 부분

■ 기타
  □ 멀티탭 → 이상한 구멍이나 USB 포트
  □ 옷걸이 고리 → 볼트 부분 구멍
```

**화장실 체크리스트 (10항목)**:
```
  □ 거울 → 양면 거울 테스트
  □ 환풍구 → 격자 사이 렌즈
  □ 칸막이 나사 → 나사 모양 카메라
  □ 휴지 거치대 → 구멍 확인
  □ 코트 훅 → 핀홀 카메라
  □ 천장 타일 → 들뜬 부분
  □ 쓰레기통 상단 → 구멍
  □ 청소용품 거치대 → 상단
  □ 비상벨 장치 → 렌즈 확인
  □ 조명 기구 → 커버 내부
```

## 11. 법적 고려사항

### 11.1 면책 조항 (앱 내 표시)
```
본 앱은 숨겨진 카메라 탐지를 보조하는 1차 스크리닝 도구입니다.

• 100% 탐지를 보장하지 않으며, 전문 탐지 장비(RF 스캐너, NLJD 등)를 대체하지 않습니다.
• 탐지 결과는 참고용이며, 의심 시 전문 업체 또는 경찰(112)에 신고하시기 바랍니다.
• Wi-Fi에 연결되지 않은 카메라, 전원이 꺼진 카메라는 탐지할 수 없습니다.
• 본 앱 사용으로 인한 직접적·간접적 손해에 대해 책임지지 않습니다.
```

## 부록: 전문 장비 vs SearCam 포지셔닝

| 항목 | 전문 탐지 장비 (RF/NLJD) | SearCam |
|------|------------------------|---------|
| 형태 | 전용 하드웨어 (수십~수백만원) | 무료 스마트폰 앱 |
| 대상 | B2B (전문 업체, 기관) | B2C (일반 사용자) |
| 정확도 | 90%+ | 70~85% |
| 접근성 | 전문가 필요 | 즉시 다운로드 |
| 핵심 차별 | 전문 정밀 탐지 | **플래시 렌즈 감지 + 교차 검증** |
| 포지셔닝 | **정밀 건강검진** | **자가 건강체크** |

> SearCam은 전문 장비의 대체재가 아니라, 일반인이 언제든 사용할 수 있는 **1차 스크리닝 도구**입니다.
