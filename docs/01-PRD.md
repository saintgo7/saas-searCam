## 2. Executive Summary

SearCam(Search + Camera)은 **스마트폰만으로 몰래카메라를 탐지하는 1차 스크리닝 앱**이다.

숙소, 화장실, 탈의실 등에서 일반인이 **30초 안에** 숨겨진 카메라 여부를 점검할 수 있도록 한다. Wi-Fi 네트워크 스캔, 플래시 Retroreflection 렌즈 감지, 자기장(EMF) 센서를 교차 검증하여 **종합 탐지율 70~75%, 오탐율 5~10%**를 목표로 한다.

### 핵심 차별점

1. **3-Layer 교차 검증**: 단일 센서가 아닌 네트워크 + 렌즈 + 자기장 복합 분석으로 오탐을 대폭 줄임
2. **플래시 Retroreflection 렌즈 감지**: 밝은 환경, 전원 OFF 카메라도 탐지 시도 (기존 앱에 없는 기능)
3. **근거 기반 결과 설명**: "위험"만 표시하지 않고, 왜 위험한지 구체적 근거를 사용자에게 제시
4. **핵심 기능 완전 무료**: 안전 기능에 페이월을 두지 않음
5. **솔직한 한계 안내**: 탐지 불가능한 상황을 명확히 고지하여 거짓 안심 방지

### 제품 포지셔닝

전문 탐지 장비(RF 스캐너, NLJD)가 "정밀 건강검진"이라면, SearCam은 **"자가 건강체크"**에 해당한다. 전문 장비를 대체하지 않으며, "없는 것보다 훨씬 나은 1차 스크리닝 도구"로 포지셔닝한다.

## 3. 문제 정의

### 3.1 몰래카메라 문제의 심각성

불법 촬영은 한국 사회의 심각한 범죄이며 매년 수천 건 이상 발생한다. 숙박업소, 공중화장실, 탈의실 등에서 초소형 카메라가 발견되는 사례가 지속적으로 보고되고 있다. 피해자는 심각한 정신적 피해를 입으며, 일반인이 스스로를 보호할 수 있는 접근 가능한 도구가 부재하다.

```
[몰래카메라 문제 구조]

  환경 위협                     피해 영향
  +-----------+                +------------------+
  | 숙박업소  |----+           | 정신적 피해      |
  | 공중화장실|    |           | 2차 유포 피해    |
  | 탈의실    |----+----->     | 사회적 불안감    |
  | 피트니스  |    |           | 일상 활동 위축   |
  | 사무공간  |----+           +------------------+
  +-----------+
                    ^
                    |
              탐지 수단 부재
              (일반인 접근 불가)
```

### 3.2 현재 솔루션의 한계

#### 전문 탐지 장비

| 장비 | 비용 | 한계 |
|------|------|------|
| RF 스캐너 | 50만~300만원 | 비용, 전문 지식 필요 |
| NLJD (비선형 접합 탐지기) | 200만~1,000만원 | 비용, 전문가 운용 필수 |
| 열화상 카메라 | 100만~500만원 | 별도 하드웨어 필요 |

일반 소비자가 숙소 체크인 시점에 전문 장비를 동원하는 것은 현실적으로 불가능하다.

#### 기존 스마트폰 앱

| 공통 약점 | 상세 |
|----------|------|
| 높은 오탐률 | 55~70%로 신뢰도 저하 |
| 수동 모드 전환 | 각 탐지 방식을 일일이 전환해야 함 |
| 결과 불투명 | "위험" 표시만 하고 근거를 설명하지 않음 |
| 핵심 기능 페이월 | 무료 버전은 기능이 제한됨 |
| 한계 미고지 | 탐지 못 하는 상황을 알리지 않아 거짓 안심 유발 |

### 3.3 시장 기회

- 글로벌 개인 보안 앱 시장은 매년 성장 중
- 한국 시장에 최적화된 경쟁 제품 부재 (기존 앱 대부분 글로벌 타겟)
- 숙박 공유 플랫폼(에어비앤비 등) 사용 증가로 몰래카메라 우려 확산
- "핵심 무료 + 솔직한 한계 고지" 전략으로 기존 앱과 차별화 가능

## 6. 사용자 스토리

### Epic 1: 빠른 안전 점검 (Quick Scan)

**US-01: 원클릭 빠른 스캔**
- As a 여행자, I want 앱을 열고 버튼 하나로 Wi-Fi 네트워크를 스캔할 수 있기를, so that 숙소 체크인 직후 30초 안에 안전을 확인할 수 있다.
- Acceptance Criteria:
  - [ ] 홈 화면에 "Quick Scan" 버튼이 최상단에 위치한다
  - [ ] 버튼 탭 후 Wi-Fi 연결 상태를 자동 확인한다
  - [ ] 스캔 완료까지 30초 이내여야 한다
  - [ ] 스캔 중 진행률(Step별 상태)을 실시간 표시한다

**US-02: Quick Scan 결과 확인**
- As a 여행자, I want 스캔 결과를 숫자 + 색상 + 한국어로 쉽게 이해할 수 있기를, so that 기술 지식 없이도 안전 여부를 판단할 수 있다.
- Acceptance Criteria:
  - [ ] 종합 위험도를 0~100 숫자와 5단계 색상(초록~빨강)으로 표시한다
  - [ ] 발견된 기기 목록을 종류별로 분류하여 보여준다 (iPhone x2, 공유기 x1 등)
  - [ ] "이상이 감지되지 않았습니다" 같은 한국어 안내 문구를 제공한다
  - [ ] Quick Scan의 한계(Wi-Fi 카메라만 점검)를 결과 화면에 안내한다

**US-03: Wi-Fi 미연결 시 대응**
- As a 사용자, I want Wi-Fi에 연결되지 않은 상태에서도 사용 가능한 탐지 모드를 안내받기를, so that 어떤 환경에서든 점검을 시작할 수 있다.
- Acceptance Criteria:
  - [ ] Wi-Fi 미연결 시 "Wi-Fi 스캔 불가" 안내와 함께 렌즈 찾기 / EMF 스캔을 제안한다
  - [ ] Layer 1을 스킵하고 Layer 2 + Layer 3만 자동 실행하는 옵션을 제공한다

### Epic 2: 정밀 통합 스캔 (Full Scan)

**US-04: 3-Layer 통합 스캔 실행**
- As a 출장 직장인, I want Wi-Fi + 렌즈 감지 + 자기장 센서를 한 번에 실행하는 Full Scan을 할 수 있기를, so that 개별 모드를 전환할 필요 없이 정밀 점검할 수 있다.
- Acceptance Criteria:
  - [ ] Full Scan 시작 시 3개 Layer가 순차/병렬로 자동 실행된다
  - [ ] 각 Layer 진행 상태를 실시간으로 표시한다
  - [ ] 전체 소요시간 약 3분 이내여야 한다
  - [ ] Layer별 개별 결과와 종합 교차 검증 결과를 모두 표시한다

**US-05: 교차 검증 결과 확인**
- As a 사용자, I want 교차 검증을 통해 오탐인지 실제 위협인지 판별된 결과를 보고 싶기를, so that 불필요한 공포 없이 정확한 판단을 할 수 있다.
- Acceptance Criteria:
  - [ ] 복수 Layer 양성 시 보정 계수가 적용된 종합 위험도를 표시한다
  - [ ] 자기장 단독 양성일 경우 "가전/금속일 가능성이 높습니다" 안내를 표시한다
  - [ ] 2개 이상 Layer 동시 양성이면 위험도를 상향 보정한다

**US-06: 근거 기반 결과 설명**
- As a 사용자, I want 왜 "위험"으로 판정되었는지 구체적 근거를 확인하고 싶기를, so that 결과를 신뢰하고 적절한 대응을 할 수 있다.
- Acceptance Criteria:
  - [ ] 각 Layer별 탐지 근거를 상세히 표시한다 (예: "MAC: 28:57:BE - Hikvision, RTSP 포트 개방")
  - [ ] 위험도 점수에 어떤 항목이 기여했는지 항목별 점수를 표시한다
  - [ ] "확인이 필요합니다"와 "카메라가 강하게 의심됩니다"를 구분하여 표시한다

### Epic 3: 렌즈 감지 (Retroreflection)

**US-07: 플래시 렌즈 찾기**
- As a 피트니스 이용자, I want Wi-Fi 없이 플래시를 켜고 카메라 렌즈를 찾을 수 있기를, so that 탈의실처럼 네트워크가 없는 환경에서도 점검할 수 있다.
- Acceptance Criteria:

## 9. 제약 조건 및 가정

### 9.1 기술적 제약

| 제약 | 상세 | 영향 | 대응 |
|------|------|------|------|
| iOS Wi-Fi 스캔 제한 | iOS는 ARP 테이블 접근 불가, MAC 주소 랜덤화(iOS 14+) | Layer 1 기능 대폭 제한 | Bonjour + 포트 스캔 중심 전략 (Phase 2) |
| IR 필터 | 일부 스마트폰은 전면 카메라에 IR 필터가 강해 IR 감지 불가 | Stage B 사용 불가 | 호환 기기 목록 안내, Stage A에 의존 |
| 자기장 정밀도 | 스마트폰 자력계는 전용 장비 대비 정밀도 낮음 | 오탐률 40~60% (단독) | 교차 검증 보정 x0.5, 보조 수단으로만 활용 |
| Retroreflection 한계 | 핀홀 카메라(0.5mm), 렌즈 코팅 카메라 감지 어려움 | 특정 카메라 유형 탐지 불가 | 한계 솔직 고지, 체크리스트 보완 |
| ARP 접근 변경 가능성 | 향후 Android 버전에서 /proc/net/arp 접근 제한 가능 | Layer 1 기능 약화 | mDNS/SSDP 대체 로직 준비 |

### 9.2 비즈니스 제약

| 제약 | 상세 |
|------|------|
| 법적 면책 필수 | "100% 탐지 보장 불가" 면책 조항을 반드시 표시해야 함 |
| 스토어 심사 정책 | 카메라/위치 권한 사용 용도를 명확히 설명해야 함 |
| 수익보다 사용자 확보 우선 | 초기에는 평점 관리와 사용자 기반 확대가 우선 |
| 1인 개발 | Phase 1은 1인 개발 기준으로 3개월 소요 |

### 9.3 가정사항

| 가정 | 검증 방법 |
|------|----------|
| 대부분의 숨겨진 카메라는 Wi-Fi에 연결되어 있다 | 경찰청 / 보안 업체 통계 참고 |
| 사용자는 스캔 시 Wi-Fi에 연결된 상태일 것이다 | 숙소/호텔 환경 기준 (Wi-Fi 제공) |
| Android 8.0+ 사용자가 타겟 시장의 95%를 차지한다 | Google Play 통계 |
| 사용자는 기술 용어를 모른다 | UX 리서치, 페르소나 기반 |
| 카메라 렌즈의 역반사 특성은 스마트폰 플래시로 감지 가능하다 | LAPD 논문 (ACM SenSys 2021) |

## 14. 부록

### 14.1 용어집

| 용어 | 설명 |
|------|------|
| ARP (Address Resolution Protocol) | IP 주소를 MAC 주소로 변환하는 네트워크 프로토콜. ARP 테이블을 통해 같은 네트워크 기기 목록을 확인 |
| mDNS (Multicast DNS) | 로컬 네트워크에서 서비스를 자동 탐색하는 프로토콜. _rtsp._tcp 등으로 스트리밍 기기를 발견 |
| SSDP (Simple Service Discovery Protocol) | UPnP 기기를 탐색하는 프로토콜 |
| OUI (Organizationally Unique Identifier) | MAC 주소 앞 3바이트로 기기 제조사를 식별하는 IEEE 표준 |
| RTSP (Real-Time Streaming Protocol) | IP 카메라에서 영상을 스트리밍하는 표준 프로토콜. 포트 554 사용 |
| ONVIF | IP 카메라 호환성 표준 프로토콜. 포트 3702 사용 |
| Retroreflection (역반사) | 빛이 들어온 방향으로 되돌아가는 반사 현상. 카메라 렌즈의 물리적 특성 |
| EMF (Electromagnetic Field) | 전자기장. 전자기기 동작 시 발생하며 자력계로 측정 |
| IR (Infrared) | 적외선. 야간 촬영 카메라에 탑재된 IR LED를 감지하는 데 사용 |
| LAPD (Laser-Assisted Photography Detection) | ACM SenSys 2021 논문에서 제안한 ToF 레이저 기반 카메라 렌즈 탐지 방식 |
| TFLite (TensorFlow Lite) | Google의 모바일/에지 디바이스용 경량 ML 프레임워크 |
| CoreML | Apple의 온디바이스 ML 프레임워크 |
| NLJD (Non-Linear Junction Detector) | 비선형 접합 탐지기. 전문 몰카 탐지 장비 |
| RF 스캐너 | 무선 주파수 스캐너. 전문 몰카 탐지 장비 |
| CameraX | Android Jetpack 카메라 라이브러리 |
| Room | Android Jetpack 로컬 데이터베이스 라이브러리 |
| Hilt | Android 의존성 주입 프레임워크 |

본 앱은 숨겨진 카메라 탐지를 보조하는 1차 스크리닝 도구입니다.
