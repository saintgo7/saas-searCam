## 1. 디자인 원칙

### 1.1 핵심 4원칙

| 원칙 | 설명 | 구현 방안 |
|------|------|----------|
| **30초 스캔** | 기술을 모르는 일반인이 30초 안에 결과를 볼 수 있어야 한다 | 앱 열기 → 큰 버튼 1회 탭 → 결과. 중간 설정/선택 단계 없음 |
| **원클릭** | 스캔 시작까지 1탭 이내 | 홈 화면 중앙 CTA 버튼, 하위 모드도 최대 2탭 |
| **공포 방지** | 오탐 시 불필요한 공포를 유발하지 않는다 | "위험" 대신 "확인 필요" 표현. 점수 + 근거를 함께 제공 |
| **거짓 안심 방지** | 탐지 안 됨 ≠ 안전을 의미하지 않음을 명확히 전달 | 결과 하단에 항상 한계 고지 문구 표시. Quick Scan 후 Full Scan 권유 |

### 1.2 보조 원칙

| 원칙 | 설명 |
|------|------|
| 근거 기반 결과 | "왜 위험한지" 구체적 이유를 항상 표시 |
| 점진적 공개 | 기본 결과 → 탭하면 상세 → 탭하면 기술 데이터 |
| 일관된 피드백 | 스캔 중 진행률, 발견 기기 수, 남은 시간을 실시간 표시 |
| 안전한 기본값 | 감도 "보통", 모든 레이어 활성화가 기본 |

## 3. 네비게이션 구조

### 3.1 Bottom Navigation

```
┌─────────────────────────────────────────────────┐
│  [홈]      [리포트]    [체크리스트]    [설정]    │
│   🏠         📋           ✅            ⚙️      │
│  Home      Reports     Checklist     Settings   │
└─────────────────────────────────────────────────┘
```

- 4개 탭, 아이콘 + 한국어 라벨
- 활성 탭: Primary 색상 아이콘 + 라벨
- 비활성 탭: Gray-400 아이콘, 라벨 숨김

### 3.2 화면 전이도

```
[스플래시]
    │
    ├─ 최초 실행 → [온보딩 1] → [온보딩 2] → [온보딩 3] → [권한 요청] → [홈]
    │
    └─ 재실행 → [홈]
                 │
                 ├─ Quick Scan 버튼 → [Quick Scan 진행] → [스캔 결과]
                 │                                            ├─ 기기 탭 → [기기 상세]
                 │                                            ├─ 리포트 저장 → [리포트 상세]
                 │                                            └─ Full Scan 하기 → [Full Scan 진행]
                 │
                 ├─ Full Scan 버튼 → [Full Scan 진행] → [렌즈 찾기] → [스캔 결과]
                 │
                 ├─ 렌즈 찾기 버튼 → [렌즈 찾기]
                 │
                 ├─ IR Only 버튼 → [IR 카메라]
                 │
                 ├─ EMF Only 버튼 → [EMF 스캔]
                 │
                 ├─ 리포트 탭 → [리포트 목록] → [리포트 상세]
                 │
                 ├─ 체크리스트 탭 → [체크리스트 선택] → [체크리스트 진행]
                 │
                 └─ 설정 탭 → [설정]
```

#### 컴포넌트

| 컴포넌트 | 사양 |
|---------|------|
| 로고 | 80dp x 80dp, 앱 아이콘 |

- 자동 전환: 1.5초 후 홈 또는 온보딩으로 이동

- 텍스트 슬라이드업: 20dp 위로, 300ms, delay 200ms

#### 상태별 UI

| 상태 | 표시 |

#### 3개 화면 콘텐츠

| 화면 | 제목 | 설명 | 일러스트 |
|------|------|------|---------|
| 1: 소개 | 30초 안전 점검 | 숙소, 화장실, 탈의실을 스마트폰으로 빠르게 점검하세요 | 스마트폰 스캔 애니메이션 |
| 2: 작동 원리 | 3중 교차 검증 | Wi-Fi 스캔 + 렌즈 감지 + 자기장 분석으로 정확도를 높입니다 | 3개 레이어 다이어그램 |
| 3: 한계 고지 | 솔직한 안내 | 전문 장비를 대체하지 않습니다. 의심 시 112에 신고해주세요 | 방패 + 경고 아이콘 |

#### 인터랙션

- 좌우 스와이프로 페이지 전환
- "건너뛰기" 탭 → 권한 요청 화면으로 즉시 이동
- "다음" 버튼 → 다음 페이지, 마지막 페이지에서는 "시작하기"로 변경
- 페이지 인디케이터: 현재 페이지 = Primary 색상 원, 나머지 = Gray-300 원

#### 애니메이션

- 페이지 전환: 수평 슬라이드, 300ms, EaseInOut
- Lottie 애니메이션: 각 페이지 진입 시 자동 재생
- 인디케이터: 크기 변환 6dp → 24dp (활성), 200ms

## 9. 접근성 기본 고려사항

### 9.1 색상 대비

- 모든 텍스트: WCAG AA 기준 (4.5:1 이상)
- 위험도 배지: 텍스트 + 배경 대비 4.5:1 이상
- 아이콘: 비텍스트 요소 3:1 이상

### 9.2 터치 타겟

- 모든 터치 가능 요소: 최소 48dp x 48dp
- 메인 CTA (Quick Scan): 120dp
- 서브 버튼: 80dp x 80dp

### 9.3 스크린 리더

- 모든 화면에 `contentDescription` 필수
- 위험도: "위험 등급 72점, 위험" 형태로 읽기
- 카메라 프리뷰: "렌즈 감지 모드 활성화, 의심 포인트 2개 발견"

### 9.4 모션 감소

- `prefers-reduced-motion` 대응
- 모션 감소 시: 게이지 애니메이션 없이 즉시 표시
- 펄스 애니메이션 → 정적 하이라이트로 대체
